# 部署说明

本文提供通用部署流程。不同服务器、云平台和域名配置会有差异，正式上线前请维护者根据实际环境补充。

## 1. 部署前准备

需要准备：

- Node.js 20 或更高版本。
- npm。
- PostgreSQL 数据库，推荐 Neon 或其他托管 PostgreSQL。
- 可用域名和 HTTPS。
- 邮件服务，当前代码使用 Azure Communication Email。
- Cloudinary 账号，用于头像上传。
- 如启用参赛证明，需配置 PDF 字体文件路径。

## 2. 环境变量

复制模板：

```bash
cp .env.example .env
```

然后补充 `.env`。不要把真实 `.env` 提交到 Git，也不要放进公开文档。

关键环境变量包括：

- `DATABASE_URL`
- `DATABASE_URL_UNPOOLED`
- `AUTH_SECRET`
- `ADMIN_REAUTH_SECRET`
- `APP_URL`
- `NEXT_PUBLIC_APP_URL`
- `AZURE_COMMUNICATION_CONNECTION_STRING`
- `AZURE_EMAIL_FROM`
- `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME`
- `CLOUDINARY_API_KEY`
- `CLOUDINARY_API_SECRET`
- `CERTIFICATE_FONT_PATH`

如果暂时不启用头像上传、邮件或参赛证明，对应功能会不可用或报错，需在上线前确认。

## 3. 本地开发部署

```bash
npm install
npm run prisma:generate
npm run prisma:migrate
npm run dev
```

默认访问：

```text
http://localhost:3000
```

开发环境的 `APP_URL` 和 `NEXT_PUBLIC_APP_URL` 通常设置为本地地址。

## 4. 生产构建

通用构建流程：

```bash
npm install
npm run prisma:generate
npm run build
```

生产环境启动：

```bash
npm run start
```

如果构建时遇到 `.next/lock`，通常说明已有 `next build` 进程正在运行。先停止旧进程，再重新构建。

## 5. 数据库迁移

开发环境：

```bash
npm run prisma:migrate
```

生产环境：

```bash
npx prisma migrate deploy
```

注意：

- 生产迁移前应备份数据库。
- 不要在生产环境随意运行会生成新迁移的命令。
- 如果使用 Neon，需要确认连接字符串包含 SSL 参数，并区分 pooled 与 unpooled 连接。

## 6. 自建服务器部署参考

常见流程：

```bash
git pull
npm install
npx prisma migrate deploy
npm run build
pm2 start npm --name "pingpong-platform" -- run start
```

更新时：

```bash
git pull
npm install
npx prisma migrate deploy
npm run build
pm2 restart pingpong-platform --update-env
```

如已有 PM2 应用名不是 `pingpong-platform`，请使用实际名称。

## 7. Nginx / 反向代理

如果使用自建服务器，建议用 Nginx 或其他反向代理：

- 对外提供 80/443。
- 将请求转发到 Next.js 应用端口。
- 配置 HTTPS 证书。
- 设置合理的上传体积限制，头像上传当前代码限制为 2MB。

示例需要根据服务器环境补充，不建议直接复制网上配置到生产。

## 8. Netlify 部署

仓库包含 `netlify.toml`：

```toml
[build]
  command = "npm run prisma:generate && npm run build"

[[plugins]]
  package = "@netlify/plugin-nextjs"
```

如果使用 Netlify：

- 在 Netlify 控制台配置所有环境变量。
- 确认 PostgreSQL 数据库可从 Netlify 访问。
- 数据库迁移建议在 CI 或维护者本地明确执行，不要依赖未确认的自动流程。
- Server Actions、PDFKit、字体文件和 Cloudinary 在 Netlify 上的运行状态需维护者确认。

## 9. 域名与 Server Actions allowed origins

`next.config.ts` 中配置了 Server Actions allowed origins。正式换域名时，需要同步检查该配置。

如果域名不在 allowed origins 中，某些 Server Actions 可能无法正常提交。

## 10. 参赛证明 PDF

参赛证明接口需要：

- `CERTIFICATE_FONT_PATH` 指向可用字体。
- 部署产物中包含字体文件。
- 运行环境支持 Node.js runtime。

当前 `next.config.ts` 已为证书接口包含 `assets/fonts` 和 PDFKit 数据文件，但生产环境仍需确认路径是否正确。

## 11. 上线检查清单

- 环境变量已配置，且没有真实密钥进入 Git。
- 数据库迁移已执行。
- 管理员账号可登录，并完成邮箱验证。
- 邮件发送可用。
- 头像上传可用。
- 比赛创建、报名、赛果确认、排行榜正常。
- 管理员后台二次验证正常。
- HTTPS 正常。
- 备份策略已准备。
- 当前仓库版本和线上版本对应关系已记录。
