# 项目结构

本项目使用 Next.js App Router。主要业务代码在 `src/app`、`src/components`、`src/lib` 和 `prisma` 中。

## 主要目录

| 路径 | 用途 | 维护注意事项 |
| --- | --- | --- |
| `src/app/` | 页面、布局、API routes 和 Server Actions | App Router 入口。新增页面时按路由目录组织。 |
| `src/app/page.tsx` | 首页 | 展示品牌、个人数据、报名比赛和近期比赛。 |
| `src/app/layout.tsx` | 全站布局 | 包含侧边栏、移动端顶部栏、页脚和 CSRF 注入组件。 |
| `src/app/matchs/` | 比赛大厅、比赛详情、创建、编辑、分组页面 | 目录名为 `matchs`，不是标准英文 `matches`，改名会影响路由。 |
| `src/app/rankings/` | 排行榜页面 | 展示 ELO、积分、荣誉榜。 |
| `src/app/profile/` | 个人中心、个人主页、资料编辑 | 与头像上传、个人战绩展示相关。 |
| `src/app/auth/` | 登录、注册、邮箱验证、密码重置 | 涉及认证和安全，修改前应重点测试。 |
| `src/app/admin/` | 管理员控制台和后台 Server Action | 包含高风险操作，应谨慎修改权限逻辑。 |
| `src/app/quick-match/` | 快速比赛页面和动作 | 用于日常对局登记、确认和 ELO 更新。 |
| `src/app/team-invites/` | 双打组队邀请页面和动作 | 服务双打比赛组队流程。 |
| `src/app/api/` | API routes | 当前包含 CSRF token、站点状态、参赛证明 PDF 导出。 |
| `src/components/` | 复用 UI 和业务组件 | 按业务域分为 auth、layout、match、quick-match、ui 等。 |
| `src/components/match/` | 比赛卡片、创建表单、赛果表单、分组管理等 | 多数比赛交互组件在这里。 |
| `src/components/match/detail/` | 比赛详情页分区组件 | 包含分组结果、个人进度、参赛者、证书导出、管理员赛果等。 |
| `src/components/layout/` | 侧栏、顶部栏、页脚、管理员模式切换 | 修改导航时重点查看这里。 |
| `src/components/security/` | CSRF 表单注入组件 | 与 Server Actions 的 CSRF 校验配合使用。 |
| `src/lib/` | 业务工具、认证、安全、ELO、Prisma、比赛逻辑 | 项目核心规则集中在这里，改动需谨慎。 |
| `src/lib/prisma.ts` | Prisma Client 单例 | 避免开发环境重复创建数据库连接。 |
| `src/lib/auth.ts` | 当前用户读取 | 多数页面和动作通过它判断登录态。 |
| `src/lib/session.ts` | Session cookie 签名与校验 | 修改会导致登录态行为变化。 |
| `src/lib/password.ts` | 密码哈希与校验 | 使用 scrypt，勿改为明文或弱哈希。 |
| `src/lib/csrf.ts` | CSRF 校验 | 所有写操作应通过该校验。 |
| `src/lib/elo.ts` | ELO 计算 | 修改会影响排名和历史数据解释。 |
| `src/lib/tournament.ts` | 分组和淘汰赛生成 | 修改前应准备比赛编排测试样例。 |
| `src/lib/doubles.ts` | 双打组队和报名相关逻辑 | 与 team-invites 页面配合。 |
| `src/lib/certificate.ts` | 参赛证明资格和身份哈希 | 涉及个人身份信息，注意隐私。 |
| `prisma/schema.prisma` | 数据库模型 | 变更必须配套迁移和测试。 |
| `prisma/migrations/` | 数据库迁移文件 | 生产环境应使用 `prisma migrate deploy`。 |
| `public/` | 静态资源 | 当前包含 robots 和协会 SVG 资源。 |
| `assets/` | 构建时使用的资源 | 当前包含证书 PDF 可用的字体文件。 |
| `middleware.ts` | 中间件 | 处理站点关闭和登录跳转。 |
| `next.config.ts` | Next.js 配置、安全响应头、Server Actions allowed origins | 域名变化时需检查 allowed origins。 |
| `netlify.toml` | Netlify 构建配置 | 若不用 Netlify，可保留作为参考。 |
| `prisma.config.ts` | Prisma CLI 配置 | 使用 `DATABASE_URL` 和 `DATABASE_URL_UNPOOLED`。 |
| `.env.example` | 环境变量模板 | 只能放变量名和示例占位，不要写真实值。 |

## 路由概览

| 路由 | 作用 |
| --- | --- |
| `/` | 首页 |
| `/auth` | 登录、注册、账号帮助 |
| `/auth/verify` | 邮箱验证 |
| `/auth/reset-password` | 重置密码 |
| `/matchs` | 比赛大厅 |
| `/matchs/create` | 创建比赛 |
| `/matchs/[id]` | 比赛详情 |
| `/matchs/[id]/edit` | 编辑比赛 |
| `/matchs/[id]/grouping` | 分组与签位管理 |
| `/rankings` | 排行榜 |
| `/profile` | 我的个人中心 |
| `/profile/edit` | 编辑个人资料 |
| `/profile/[id]` | 公开个人主页 |
| `/quick-match` | 快速比赛 |
| `/team-invites` | 双打组队邀请 |
| `/admin` | 管理员控制台 |
| `/closed` | 站点关闭提示页 |
| `/api/csrf-token` | CSRF token 获取 |
| `/api/site-status` | 站点状态查询 |
| `/api/matchs/[id]/certificate` | 参赛证明 PDF 导出 |

## 命名提醒

- 比赛路由使用 `matchs`，这是当前代码真实路由。不要在文档或链接中误写成 `matches`。
- 用户可见功能和数据库模型不是一一对应。模型中存在奖励、评价、排行榜缓存等结构，但当前代码没有看到完整页面，维护者应确认是否为预留功能。
