# 运维说明

本文面向未来维护者，说明日常维护、排障和交接时需要关注的事项。

## 1. 日常检查

建议定期检查：

- 网站首页、比赛大厅、排行榜和登录页是否可访问。
- 数据库连接是否稳定。
- 邮件服务是否能发送验证邮件和管理员验证码。
- Cloudinary 头像上传是否正常。
- 管理员后台是否能完成二次验证。
- 是否存在异常 500 错误或构建失败。

## 2. 数据备份

数据库是最重要的资产。至少应备份：

- 用户信息。
- 比赛、报名、赛果。
- ELO 历史。
- 积分流水。
- 分组结果。
- 审计日志。

如果使用 Neon，可以考虑平台自带的备份、分支或时间点恢复能力。具体策略需维护者根据当前套餐确认。

## 3. 数据库迁移

开发环境使用：

```bash
npm run prisma:migrate
```

生产环境使用：

```bash
npx prisma migrate deploy
```

迁移前建议：

- 备份数据库。
- 阅读迁移 SQL。
- 在测试库验证。
- 记录执行时间、执行人和对应 Git commit。

不要在生产环境直接修改 schema 后“试试看”。

## 4. 日志排查

自建服务器常见命令：

```bash
pm2 logs pingpong-platform
pm2 status
```

如果应用名不同，请使用实际 PM2 名称。

需要重点关注：

- 登录失败和邮件发送失败。
- Prisma 数据库连接错误。
- 证书 PDF 字体路径错误。
- Server Actions 提交失败。
- 管理员后台操作异常。

## 5. 管理员账号

管理员账号具有用户管理、批量操作、站点开关和赛果管理能力。交接时应确认：

- 当前管理员有哪些人。
- 是否存在离任管理员账号。
- 管理员邮箱是否可收验证码。
- 是否启用了信任设备。
- 管理员操作是否能在审计日志中追踪。

离任管理员应及时降权或封禁。

## 6. 站点关闭

管理员后台包含站点开关。关闭后普通用户会被引导到关闭页面，管理员仍可进入后台。

适合在以下情况使用：

- 数据库迁移。
- 紧急修复。
- 规则调整。
- 发现异常写入或攻击迹象。

站点关闭前，最好在社团群里同步原因和预计恢复时间。

## 7. 域名、HTTPS 与反向代理

如果使用自建服务器：

- 域名解析应指向正确服务器。
- HTTPS 证书应定期续期。
- Nginx 或其他反向代理应转发到 Next.js 服务端口。
- 上传体积、超时、缓存策略需要与应用行为匹配。

如果使用 Netlify 或其他平台：

- 域名和 HTTPS 通常由平台托管。
- 仍需确认环境变量、构建日志和 Server Actions allowed origins。

## 8. 依赖更新

依赖更新前建议：

1. 新建分支。
2. 更新依赖。
3. 运行 `npm run lint`。
4. 运行 `npm run build`。
5. 用测试账号验证登录、报名、赛果确认、管理员后台。
6. 再合并和部署。

不要在比赛当天临时大版本升级框架或 Prisma。

## 9. 比赛数据维护

比赛进行中不要随意直接改数据库。优先使用系统功能：

- 报名和退赛走页面按钮。
- 分组走分组管理页。
- 赛果走登记、确认、否决或纠错流程。
- 移除参赛者走管理员功能。

如果确实需要数据库级修复，应记录原因、SQL、执行人和备份点。

## 10. 常见故障

### 登录后又跳回登录页

可能原因：

- `AUTH_SECRET` 不一致或缺失。
- cookie 未正确写入。
- 用户邮箱未验证。
- 用户被封禁。
- 部署环境 HTTPS / cookie secure 配置异常。

### 邮件发不出去

可能原因：

- Azure Communication Email 环境变量缺失。
- 发件地址未配置或未验证。
- 第三方邮件服务限流。

### 构建卡住或提示 `.next/lock`

可能已有 `next build` 正在运行。先停止旧构建进程，再重新执行：

```bash
npm run build
```

### 数据库连接失败

检查：

- `DATABASE_URL`
- `DATABASE_URL_UNPOOLED`
- SSL 参数
- 数据库是否允许当前部署环境访问
- 是否达到连接数限制

### PDF 证明导出失败

检查：

- `CERTIFICATE_FONT_PATH`
- 字体文件是否存在
- 部署环境是否包含 `assets/fonts`
- 用户是否满足导出资格

## 11. 交接建议

每次换届至少交接：

- 当前部署平台。
- 服务器或平台账号归属。
- 数据库平台和备份方式。
- 管理员账号列表。
- 域名和 HTTPS 管理方式。
- 邮件服务和 Cloudinary 账号归属。
- 最近一次部署 commit。
- 已知问题和暂未启用功能。
